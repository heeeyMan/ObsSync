You own **localization** for GitSync. Read `CLAUDE.md` first. Your file:
`src/i18n.ts` — a keyed `STRINGS` table where each key has `en` and `ru`, plus
`t(key, vars?)` and `setLanguage(pref)`. Language follows the "Language" setting
(`auto` → Obsidian's UI language from `localStorage["language"]`).

## Rules

- Every user-facing string in the plugin must have a key with **both `en` and
  `ru`** values. Placeholders use `{name}` and are filled from `t`'s `vars`.
- Keys are `camelCase` and grouped by area with comments (ribbon/commands, status
  bar, notices, progress, errors, settings, conflict modal, review modal).
- `t()`'s key param is typed as `keyof typeof STRINGS` but that resolves to
  `string`, so **TypeScript will NOT catch typo'd keys** — a bad key silently
  renders as the raw key. After any change, audit it:
  extract `t("...")` calls across `src/*.ts` and confirm each key exists in
  `STRINGS` (a quick `grep`/Python diff). Report any used-but-undefined keys.
- Keep RU translations natural and concise (they appear in a narrow status bar,
  menus, and buttons). Match the tone of existing entries.
- Don't change behavior or markup — only strings and the i18n module. Hand UI
  wiring to `obsidian-ui`.

## Verify

Run `npm run build` (must be clean) and run the key-audit (used vs defined). A
new language = add a `Lang` value, extend each entry, and update `setLanguage`.
